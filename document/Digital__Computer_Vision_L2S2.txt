Apply and explain histogram equalization with the method CLAHE (Contrast Limited Adaptive Histogram
Equalization) of OpenCV

Instead of applying the same histogram calculation to the whole image, CLAHE operates locally in small regions
(tiles). Standard Histogram Equalization can cause over-bright areas and excessive noise in images with large
intensity variations. CLAHE avoids this by limiting the contrast amplification.

Algorithm :
1. Divide Image into Small Regions (Tiles) : The image is split into small non-overlapping blocks (tiles),
typically 8x8 or 16x16 pixels.
2. Apply Histogram Equalization to Each Tile : Histogram Equalization is applied separately to each tile.
3. Clip the Histogram (Limit Contrast Amplification) : A contrast limit (clip limit) is applied to avoid over-
enhancing noise. Pixel intensities are redistributed to prevent over-saturation :
4. Smooth Transition Between Tiles : To avoid artificial edges, CLAHE blends neighboring tiles using bilinear
interpolation.

Edges

Edges in an image are boundaries where intensity or color changes significantly, usually at object
borders. These edges help identify structures and shapes, ideally forming a continuous outline around
objects.

Detecting edges using Sobel

Edge detection is used to identify object boundaries. The Sobel operator is a gradient-based edge
detection method that calculates the rate of intensity change in an image. It applies two convolution
kernels to detect edges in the X and Y directions separately.




 To get the final edge map, we can combine both Sobel X and Sobel Y using Weighted Sum or Magnitude
 Formula:
When to use Sobel edge detection ?

1. Best for detecting vertical and horizontal edges.
2. Less sensitive to noise than other gradient operations.
3. Works well for structured images (text, roads, objects).

Not the best for fine details—Canny edge detection may work better in some cases.
Sobel edge detection example example
import cv2

# Load an image

sobel_y = cv2.Sobel(image, cv2.CV_64F, 0, 1, ksize=3)

Canny edge detection

Canny Edge Detection is a multi-step algorithm developed by John F. Canny in
1986. It is one of the most widely used edge detection techniques because it
provides clear and well-defined edges while reducing noise.

When to Use Canny Edge Detection?
1. Best for detecting well-defined edges
2. Works well for shape detection, segmentation, object tracking
3. Better than Sobel for real-world images with noise
Canny edge detection algorithm

Canny Edge Detection follows a 4-step process:
1.️ Noise Reduction (Gaussian Blur) : The image is first smoothed using a Gaussian Blur to remove noise.
This avoids detecting false edges caused by random intensity variations.
2️. Compute Gradient (Sobel Filter): Compute gradients in X and Y directions using Sobel operators.
Detects regions with rapid intensity change (edges).
3️. Non-Maximum Suppression: Every pixel is checked if it is a local maximum in its neighborhood in the
direction of gradient. If so, it is considered for next stage, otherwise, it is suppressed ( put to zero).
4️. Hysteresis Thresholding : Classifies pixels into strong edges, weak edges, and non-edges. For this, we need two
threshold values, minVal and maxVal.
Strong edges: above the maxVal ,Keep them in the final image.
Weak edges: between minVal and maxVal. Only kept if connected to strong edges.
Non-edges: below minVal - removed.
Canny edge detection example

Laplacian Edge Detection

Laplacian edge detection is a second-order derivative method used to
detect edges in an image. Unlike Sobel or Canny, which use first-order
derivatives (gradients), the Laplacian detects edges based on changes in
the gradient.

Key Features:
1. Uses the Laplacian operator (∇²) to detect edges.
2. Sensitive to noise because it uses second derivatives.
3. Highlights edges in all directions at once.
Laplacian Operator

Tresholding

Thresholding is a technique used in image processing to convert a grayscale image into a binary image
(black & white). It is useful for object segmentation, edge detection, and shape analysis.

In thresholding, each pixel is compared to a threshold value:
If the pixel value is greater than the threshold → it is set to white (255).
If the pixel value is less than the threshold → it is set to black (0).
Simple Thresholding in OpenCV

Basic thresholding applies a fixed threshold to all pixels. The function cv.threshold is used to apply the
thresholding. The first argument is the source image, which should be a grayscale image. The second
argument is the threshold value which is used to classify the pixel values. The third argument is the
maximum value which is assigned to pixel values exceeding the threshold. OpenCV provides different
types of thresholding which is given by the fourth parameter of the function.

cv.THRESH_BINARY                                   white for pixel > treshold, else black
cv.THRESH_BINARY_INV                               Inverted binary (black & white swapped)
cv.THRESH_TRUNC                                    Values > treshold set to treshold, rest unchanged
cv.THRESH_TOZERO                                   Values < treshold set to 0, rest unchanged
cv.THRESH_TOZERO_INV                               Inverted ToZero
cv.THRESH_OTSU                                     Otsu’s tresholding
Simple Thresholding example

Adaptive Thresholding


If an image has uneven lighting, simple thresholding might not work well. In Adaptive thresholding the
algorithm determines the threshold for a pixel based on a small region around it. So we get different thresholds
for different regions of the same image which gives better results for images with varying illumination.

In addition to the parameters described above, the method cv.adaptiveThreshold takes three input parameters:

The adaptiveMethod decides how the threshold value is calculated:

cv.ADAPTIVE_THRESH_MEAN_C: The threshold value is the mean of the neighbourhood area minus the
constant C.
cv.ADAPTIVE_THRESH_GAUSSIAN_C: The threshold value is a gaussian-weighted sum of the
neighbourhood values minus the constant C.

The blockSize determines the size of the neighbourhood area and C is a constant that is subtracted from the
mean or weighted sum of the neighbourhood pixels.
Adaptive Thresholding example

import cv2
from matplotlib import pyplot as plt

 for i in range(4):

Otsu's Thresholding


Otsu’s method determines the optimal threshold automatically. Otsu's method determines an optimal global
threshold value from the image histogram. Otsu's algorithm split the image into two classes:
Pixels with values ≤ T (background).
Pixels with values > T (foreground).

then select the threshold value that minimizes the intra-class variance (or maximizes inter-class variance).
 Otsu tresholding is effective for Bi-modal Histograms

In OpenCV, the cv.threshold() function is used, where cv.THRESH_OTSU is passed as an extra flag. The
threshold value can be chosen arbitrary. The algorithm then finds the optimal threshold value which is returned
as the first output.
Otsu's Thresholding example 1/2

Finding and drawing contours in images

Contours are curves that join continuous points along a boundary of objects with the same color or
intensity in an image. They are useful in:

1. Object detection.
2. Shape analysis.
3. Image segmentation.
4. Measuring object properties (size, area, etc.).

Since contours are computed only on binary images we have first convert the image to grayscale then
apply thresholding or Canny edge detection.
Finding and drawing contours functions in OpenCV

To find contours :
contours, hierarchy = cv2.findContours(image, mode, method)
image → Binary image (must be thresholded or edge-detected).
mode → Defines contour retrieval:
          cv2.RETR_EXTERNAL → Retrieves only the outermost contours.
          cv2.RETR_TREE → Retrieves all contours in a hierarchical structure.
method → Contour approximation method:
          cv2.CHAIN_APPROX_SIMPLE → Stores only important points, reducing memory usage.
          cv2.CHAIN_APPROX_NONE → Stores all boundary points (more data, but unnecessary).

To draw contours :
cv2.drawContours(image, contours, contourIndex, color, thickness)
image → The image where contours are drawn.
contours → The list of contour points.
contourIndex → Index of the contour to draw (-1 draws all contours).
color → The color of the contour (e.g., (0, 255, 0) for green).
thickness → Line thickness (use -1 to fill the contour).
Finding and drawing contours example

import cv2

# Load the image
image = cv2.imread('../Images/opencv-logo.png')

# Convert to grayscale and find edges
gray = cv2.cvtColor(image, cv2.COLOR_BGR2GRAY)
edges = cv2.Canny(gray, 100, 200)

# Find contours from edges
contours, _ = cv2.findContours(edges, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)

# Draw contours on the original image
cv2.drawContours(image, contours, -1, (255, 255, 255), 2)

# Show the result
cv2.imshow('Contours', image)
cv2.waitKey(0)
cv2.destroyAllWindows()
Contour Features

Contours provide valuable shape and structure information about objects in an image shape
classification (circles, rectangles, irregular shapes) and size meazurement. With OpenCV is posible to
extract :

1. Contour Area : Measures the area enclosed by the contour.
2. Contour Perimeter : Measures the length of the contour boundary.
3. Contour Centroid :Finds the center (centroid) of a contour using image moments.
4. Bounding Box : Finds the smallest rectangle that encloses the contour.
5. Minimum Enclosing Circle : Finds the smallest circle enclosing the contour.
6. Aspect Ratio (Width-to-Height Ratio) : Measures how elongated an object is.
7. Convex Hull : Finds the smallest convex shape enclosing the contour.

Hough Transform

Hough Transform is a powerful technique used to detect geometric shapes, if you can represent that
shape in a mathematical form, like lines and circles, even if they are incomplete or noisy.

Types of Hough Transform in OpenCV :
1. Hough Line Transform – Used to detect straight lines.
2. Hough Circle Transform – Used to detect circles.
Hough Line Transform – how it works ?

A straight line in the image space (Cartesian) can be represented as:
                               y=mx+c
However, this equation has problems when m (slope) is infinite (vertical lines). So, we use an
alternative parameterization:
                               ρ=xcosθ+ysinθ
Where:
ρ = perpendicular distance from the origin to the line
θ = angle between the x-axis and the perpendicular line




Key Idea:
Instead of detecting lines directly, we transform all edge points into the (ρ, θ) space, where they form a
curve. Points that lie on the same line in image space will intersect at a common (ρ, θ) point in Hough
space.
Hough Line Transform Algorithm

1. Binarisation : Input image should be a binary image, so apply threshold or use canny edge detection
before applying hough transform
2. Create a 2D Accumulator Matrix : We create a 2D matrix called the Hough Accumulator where
rows represent values of ρ, columns represent values of θ (from 0° to 180°). Each cell in this matrix is
initialized to zero.
3. Vote in the Hough Space : For each edge point (x, y) in the image we calculate (ρ, θ). we increment
the corresponding (ρ, θ) cell in the accumulator.
4. Find Peaks in the Accumulator : The accumulator now contains peaks where many edge points
contribute to the same (ρ, θ). These peaks correspond to lines in the image.
5. Convert (ρ, θ) Back to Cartesian : After detecting peaks in the Hough space, we convert them back
into lines. The detected (ρ, θ) values are converted back to line equations in Cartesian space and drawn
on the image.
Hough Line Transform in OpenCV

OpenCV’s propose cv2.HoughLines() function with parameters
Input binarised image
Distance resolution of the accumulator (step size for ρ in pixels).
Angular resolution (step size for θ in radians).
Threshold (minimum votes needed to consider it a line).
Hough Line Transform example 1/2



import cv2
import numpy as np

# Load the image in grayscale
image = cv2.imread('../Images/sudoku.png')
gray = cv2.cvtColor(image, cv2.COLOR_BGR2GRAY)
edges = cv2.Canny(gray, 50, 150)

# Detect lines using Hough Transform
lines = cv2.HoughLines(edges, 1, np.pi / 180, 150)
Hough Line Transform example 2/2

# Draw the detected lines
for line in lines:
  rho, theta = line[0]
  a = np.cos(theta)
  b = np.sin(theta)
  x0 = a * rho
  y0 = b * rho
  x1 = int(x0 + 1000 * (-b))
  y1 = int(y0 + 1000 * (a))
  x2 = int(x0 - 1000 * (-b))
  y2 = int(y0 - 1000 * (a))
  cv2.line(image, (x1, y1), (x2, y2), (0, 255, 0), 2)

# Show the result
cv2.imshow("Hough Line Transform", image)
cv2.waitKey(0)
cv2.destroyAllWindows()
Hough Circle Transform

A circle can be described by:




  Instead of detecting lines, we find circles by mapping points to a 3D accumulator space. From equation,
  we can see we have 3 parameters, so we need a 3D accumulator for hough transform, which would be
  highly ineffective. So OpenCV uses more trickier method, Hough Gradient Method which uses the
  gradient information of edges.
Hough Circle Transform example

import cv2
import numpy as np

img = cv2.imread('../Images/opencv-logo.png', cv2.IMREAD_GRAYSCALE)

cimg = cv2.cvtColor(img,cv2.COLOR_GRAY2BGR)
circles = cv2.HoughCircles(img,cv2.HOUGH_GRADIENT,1,20,
                    param1=50,param2=30,minRadius=0,maxRadius=0)
circles = np.uint16(np.around(circles))
for i in circles[0,:]:
   # draw the outer circle
   cv2.circle(cimg,(i[0],i[1]),i[2],(0,255,0),2)
   # draw the center of the circle
   cv2.circle(cimg,(i[0],i[1]),2,(0,0,255),3)

cv2.imshow('detected circles',cimg)
cv2.waitKey(0)
cv2.destroyAllWindows()
